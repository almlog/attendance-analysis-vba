## 1. システム概要

### 1.1 このシステムでできること

✅ **毎日の勤怠チェックが3分で完了**
- 勤怠システムからCSVをダウンロード
- ボタン2回クリックで未入力者に通知
- LINE WORKSで管理者チャンネルに自動投稿

✅ **未入力者が一目でわかる**
- 緊急度別に色分け表示（🔴緊急/🟡要注意/🟢確認）
- 何日前から未入力かが自動計算される
- 複数日の未入力も分かりやすく表示

✅ **通知履歴を自動記録**
- いつ、誰に、何回通知したかが記録される
- 過去の対応状況を追跡可能

### 1.2 システム構成

```
あなたのPC（Excel）
    ↓
LINE WORKS 管理者チャンネル
    ↓
部門管理者が確認
    ↓
各社員に声掛け
```

### 1.3 利用環境

- **必要なソフト**: Microsoft Excel 2016以降
- **必要な権限**: LINE WORKS 管理者権限
- **ネット接続**: 必須（通知送信時のみ）
- **推奨ブラウザ**: Edge / Chrome（LINE WORKS管理画面用）

## 2. 初回セットアップ

### 2.1 準備するもの

- [ ] 勤怠管理.xlsm ファイル
- [ ] LINE WORKS アカウント（管理者権限）
- [ ] 管理者チャンネル（事前作成済み）

### 2.2 セットアップ手順（所要時間: 10分）

#### Step 1: LINE WORKS Webhook URLの取得

**画面操作手順**:

```
1. LINE WORKS 管理画面にログイン
   https://admin.worksmobile.com

2. 左メニューから [Bot] をクリック

3. [Bot追加] ボタンをクリック

4. Bot情報を入力
   - Bot名: 勤怠リマインダー
   - 説明: 勤怠未入力者への自動通知Bot
   - アイコン: お好みの画像

5. [Webhook] タブに移動

6. [Webhook URL発行] をクリック

7. 発行されたURLをコピー
   （例: https://talk.worksmobile.com/xxxxxxxx）

8. 管理者チャンネルにBotを追加
   - チャンネル > 設定 > メンバー管理
   - [Bot追加] で 勤怠リマインダー を選択
```

**重要**: このURLは外部に漏らさないでください！

---

#### Step 2: Excelファイルの初期設定

**1. VBAプロジェクトへのアクセス許可**

```
Excel を起動
  ↓
ファイル > オプション
  ↓
トラストセンター > トラストセンターの設定
  ↓
マクロの設定
  ↓
☑ VBAプロジェクトオブジェクトモデルへのアクセスを信頼する
  ↓
OK で閉じる
  ↓
Excel を再起動
```

⚠️ **この設定をしないと、VBAコードが動作しません！**

---

**2. 初期セットアップの実行**

```
1. 勤怠管理.xlsm を開く

2. [マクロを有効にする] をクリック

3. Alt + F11 でVBエディタを開く

4. 左の [VBAProject] で Module1_Setup をダブルクリック

5. コード内の InitialSetup の行にカーソルを置く

6. F5 キーを押して実行

7. "セットアップ完了！" のメッセージを確認

8. Excel画面に戻る（Alt + F11）
```

---

**3. Webhook URLの設定**

```
1. [設定] シートタブを右クリック

2. [再表示] を選択
   （非表示になっている場合）

3. B1セルをクリック

4. Step 1でコピーしたWebhook URLを貼り付け

5. Enter キーで確定

6. [設定] シートタブを右クリック > [非表示]

7. 保存（Ctrl + S）
```

✅ これで初期セットアップ完了です！

---

### 2.3 動作確認

**テスト送信をしてみましょう**

```
1. [勤怠未入力] シートを開く

2. [📥 CSV読込] ボタンの横にある
   [📤 管理者通知] ボタンをクリック
   （まだCSVを読み込んでいないので、エラーが出るかもしれません）

3. サンプルCSVがある場合は読み込んでテスト送信

4. LINE WORKS の管理者チャンネルで
   メッセージが届いていることを確認
```

**✅ メッセージが届いたら成功です！**

#### ステップ3: 未入力者を確認

**確認内容**:

```

   → 早めの声掛けが必要

🟢 確認（1-2日未入力）
   → 翌日以降も未入力なら対応
```

**対応優先順位の判断**:

| 未入力日数 | 緊急度 | 対応方法 | 対応者 |

| 3-4日 | 🟡 高 | 当日中に声掛け | 上長 |
| 1-2日 | 🟢 中 | 翌日継続なら対応 | 管理者 |

### 4.2 CSV読込エラー

#### エラー1: 「ファイルが選択されていません」

**原因**: ファイル選択ダイアログで [キャンセル] をクリックした

**対処方法**:
```
1. もう一度 [📥 CSV読込] ボタンをクリック
2. 正しいCSVファイルを選択
3. [開く] をクリック
```

---

#### エラー2: 「CSV形式が正しくありません」

**原因**: 
- CSVファイルが破損している
- 必須列が不足している
- 文字コードが不正

**対処方法**:
```
【確認1】CSVファイルをメモ帳で開く
  1. CSVファイルを右クリック
  2. [プログラムから開く] > [メモ帳]
  3. 1行目（ヘッダー）に以下があるか確認
     社員番号,氏名,日付,曜日区分,届出内容,コメント,出勤時刻,退勤時刻

【確認2】勤怠システムで再出力
  1. 勤怠システムにログイン
  2. CSVを再度ダウンロード
  3. 新しいファイルで再試行

【確認3】文字コード変換
  1. メモ帳でCSVを開く
  2. [ファイル] > [名前を付けて保存]
  3. 文字コード: UTF-8 を選択
  4. 保存して再試行
```

---

#### エラー3: 「〇行目: 日付形式が正しくありません」

**原因**: 日付列に不正な値が含まれている

**対処方法**:
```
【警告の場合】
  → 該当行はスキップされ、処理は継続されます
  → 勤怠システムでデータを修正後、再出力を推奨

【エラーの場合】
  1. CSVファイルをExcelで開く
  2. 該当行の日付列を確認
  3. 手動で修正（yyyy/mm/dd形式に）
  4. CSV形式で保存
  5. 再度読み込み
```

---

### 4.3 通知送信エラー

#### エラー1: 「Webhook URLが設定されていません」

**原因**: 設定シートにWebhook URLが未入力

**対処方法**:
```
1. [設定] シートタブを右クリック > [再表示]
2. B1セルにWebhook URLが入力されているか確認
3. 空欄の場合、LINE WORKS管理画面でURLを再取得
4. B1セルに貼り付け
5. 保存（Ctrl + S）
6. [設定] シートを右クリック > [非表示]
7. 再度 [📤 管理者通知] をクリック
```

---

#### エラー2: 「送信エラー: HTTP 404 Not Found」

**原因**: Webhook URLが無効または削除された

**対処方法**:
```
1. LINE WORKS 管理画面にログイン
2. [Bot] > [勤怠リマインダー] を選択
3. [Webhook] タブを開く
4. Webhook URLが有効か確認
5. 無効の場合、新しいURLを発行
6. Excelの設定シートに新URLを貼り付け
7. 再度送信
```

---

#### エラー3: 「送信エラー: HTTP 429 Too Many Requests」

**原因**: 短時間に送信しすぎてレート制限に達した

**対処方法**:
```
1. 5分待つ
2. 再度送信

【予防策】
- 1日の送信回数を3回以内に抑える
- テスト送信を繰り返さない
```

---

#### エラー4: 「ネットワークエラー」

**原因**: インターネット接続が切断されている

**対処方法**:
```
【確認1】ネット接続
  1. ブラウザで https://www.google.com を開く
  2. 開かない場合、ネットワーク管理者に連絡

【確認2】プロキシ設定
  1. 社内プロキシ経由の場合、設定を確認
  2. IT部門に確認

【確認3】ファイアウォール
  1. セキュリティソフトがブロックしていないか確認
  2. IT部門に除外設定を依頼
```

---

### 4.4 VBAエラー

#### エラー1: 「VBAプロジェクトへのアクセスが拒否されました」

**原因**: VBAプロジェクトへのアクセスが許可されていない

**対処方法**:
```
1. Excel を開く
2. ファイル > オプション
3. トラストセンター > トラストセンターの設定
4. マクロの設定
5. ☑ VBAプロジェクトオブジェクトモデルへのアクセスを信頼する
6. OK
7. Excel を再起動
```

**セキュリティ注意**: この設定は必要な時のみ有効にし、通常時はOFFを推奨

---

#### エラー2: 「実行時エラー '9': インデックスが有効範囲にありません」

**原因**: シートが削除されているか、名前が変更されている

**対処方法**:
```
【確認】必要なシートが存在するか
  - 勤怠未入力
  - 社員マスタ
  - 通知履歴
  - 設定

【復旧方法】
  1. バックアップファイルから復元
  2. または InitialSetup を再実行
```

### 4.5 パフォーマンス問題

#### 問題1: CSV読込が10秒以上かかる

**原因**: データ量が多い、またはPCスペック不足

**対処方法**:
```
【対策1】古いデータの削除
  1. 勤怠システムで対象期間を絞る
  2. 直近3日分のみ出力

【対策2】Excelの最適化
  1. 他のExcelファイルを閉じる
  2. 不要なアプリケーションを終了
  3. PC再起動

【対策3】IT部門に相談
  - PCスペックアップグレード検討
```

---

#### 問題2: Excel起動が遅い

**原因**: ファイルサイズが大きい

**対処方法**:
```
【確認】ファイルサイズ
  1. 勤怠管理.xlsm を右クリック > プロパティ
  2. サイズが10MB以上の場合、整理が必要

【対策】通知履歴の削除
  1. 通知履歴シートを開く
  2. 古いデータ（6ヶ月以上前）を削除
  3. 保存

【対策2】新規ファイル作成
  1. InitialSetup を新しいExcelファイルで実行
  2. 社員マスタのみコピー
  3. 古いファイルはアーカイブ
```

## 5. FAQ（よくある質問）

### 5.1 基本的な使い方

**Q1: 毎日通知を送る必要がありますか？**

A: 推奨は毎日ですが、週2-3回でも効果はあります。ただし、未入力が長期化すると記憶が曖昧になるため、できるだけ頻繁に送ることをおすすめします。

---

**Q2: 通知を送る時間は決まっていますか？**

A: 特に決まりはありませんが、**9:00（始業直後）**が推奨です。理由は以下の通り：
- 管理者がチェックしやすい
- 社員が当日中に対応できる
- 夕方だと翌日対応になりやすい

---

**Q3: 休日も通知を送る必要がありますか？**

A: いいえ、営業日のみで問題ありません。休日明けの月曜日に前週分をまとめて送信するのも効果的です。

---

### 5.2 通知内容について

**Q4: 通知メッセージの内容を変更できますか？**

A: はい、変更可能です。ただしVBAコードの編集が必要です。

```
変更手順:
1. Alt + F11 でVBエディタを開く
2. Module3_Notification を開く
3. GenerateManagerMessage 関数を編集
4. 変更後、保存して動作確認
```

変更が不安な場合は、IT部門またはシステム開発者に依頼してください。

---

**Q5: 特定の社員を通知対象から除外できますか？**

A: 現バージョンでは個別除外機能はありません。将来のバージョンで実装予定です。

**暫定対処法**:
- CSV出力後、該当社員の行を手動で削除してからExcelに読み込む

---

**Q6: 絵文字（🔴🟡🟢）が正しく表示されません**

A: LINE WORKSアプリのバージョンが古い可能性があります。

```
対処方法:
1. LINE WORKSアプリを最新版にアップデート
2. PC版の場合、ブラウザ版で確認
3. それでも表示されない場合、IT部門に相談
```

---

### 5.3 社員マスタについて

**Q7: 社員マスタのLINEWORKSメールはどこで確認できますか？**

A: LINE WORKS管理画面で確認できます。

```
1. LINE WORKS 管理画面にログイン
2. 左メニュー > [メンバー]
3. 該当社員を検索
4. プロフィール > メール を確認
```

---

**Q8: 社員マスタの入力は必須ですか？**

A: **Phase 1（現バージョン）では任意**です。管理者チャンネル通知のみ利用する場合は不要です。

**Phase 3（個別トーク通知）では必須**になります。早めに入力しておくことを推奨します。

---

**Q9: 退職者の社員マスタはどうすれば良いですか？**

A: 削除せず、LINEWORKSメール列を空欄にしてください。理由：
- 過去の通知履歴との整合性を保つため
- 将来的な統計分析に活用できるため

---

### 5.4 エラー・トラブル

**Q10: 「マクロを有効にする」が表示されません**

A: Excelのセキュリティ設定でマクロが無効化されている可能性があります。

```
対処方法:
1. ファイル > オプション
2. トラストセンター > トラストセンターの設定
3. マクロの設定
4. ○ 警告を表示してすべてのマクロを無効にする
   を選択
5. OK
6. Excelを再起動
```

---

**Q11: 突然動かなくなりました**

A: 以下の順で確認してください。

```
【確認1】Excelの再起動
  1. ファイルを保存
  2. Excelを完全に終了
  3. 再度ファイルを開く

【確認2】PCの再起動
  1. 作業中のファイルを保存
  2. PCを再起動
  3. 再度ファイルを開く

【確認3】バックアップから復元
  1. 共有フォルダのバックアップを確認
  2. 最新のバックアップファイルを開く
  3. VBAコードをインポート（必要に応じて）

【確認4】IT部門に連絡
  - 上記で解決しない場合
```

---

### 5.5 セキュリティ

**Q12: Webhook URLが流出したらどうなりますか？**

A: 悪意のある第三者が管理者チャンネルにメッセージを送信できてしまいます。

```
【対処方法】
1. LINE WORKS管理画面にログイン
2. Bot > 勤怠リマインダー
3. Webhook URLを再発行
4. Excelの設定シートに新URLを設定
5. 流出経路を特定し、再発防止策を実施
```

---

**Q13: 社員情報が外部に送信されることはありますか？**

A: いいえ、ありません。このシステムは以下のセキュリティ設計です：
- すべての処理はローカルPC（Excel）内で完結
- LINE WORKSへ送信されるのはメッセージのみ
- 社員番号や個人情報は送信されません

## 6. 用語集

| 用語 | 説明 |
|------|------|
| **CSV** | Comma-Separated Values。カンマ区切りのテキストファイル形式。勤怠データの出力形式 |
| **Webhook** | アプリ間でデータを送受信する仕組み。LINE WORKSにメッセージを送信するために使用 |
| **Bot** | 自動化されたプログラム。勤怠リマインダーBotがメッセージを送信 |
| **VBA** | Visual Basic for Applications。Excelのマクロプログラミング言語 |
| **社員マスタ** | 社員の基本情報を管理するデータベース。社員番号とLINEWORKSメールの紐付けに使用 |
| **管理者チャンネル** | LINE WORKSのグループチャット。部門管理者が参加し、通知を受け取る |
| **未入力日数** | 対象日から現在までの経過日数。日数が多いほど優先度が高い |
| **エスカレーション** | 段階的に対応を強化すること。日数経過に応じて通知方法を変える |
| **レート制限** | 一定時間内のAPI呼び出し回数制限。LINE WORKSでは600リクエスト/分 |
